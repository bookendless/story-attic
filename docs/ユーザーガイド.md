# StoryAttic ユーザーガイド

> 屋根裏の書斎で、柔らかなランプの灯りのもとに物語を紡ぐ。

StoryAttic（ストーリーアティック）は、日本語小説の執筆に特化した Windows 向けデスクトップアプリです。プロジェクト管理・リッチテキストエディタ・執筆支援パネル・AI 思考パートナーまで、創作に必要なものをひとつの画面にまとめています。

このガイドは、これから StoryAttic を使う「書き手」のための解説です。インストール直後から、最初の一話を書き上げ、作品を育てていくまでの流れに沿って機能を紹介します。

## 1. クイックスタート

StoryAttic は、インストーラーから入れてすぐに使えるネイティブアプリです。アカウント登録やインターネット接続は不要です（AI 機能を使う場合のみ、各社の API キーが必要になります）。

最初の一話を書くまでの流れは次のとおりです。

1. **StoryAttic を起動する。** 最初に「ホーム画面」が開きます。
2. **作品を作る。** 画面中央の「**＋ 新規作成**」カードをクリックし、作品タイトルを入力します。
3. **ワークスペースに入る。** 作成と同時に編集画面（ワークスペース）が開きます。
4. **章とエピソードを作る。** 左の「目次」または「章立て」パネルから、章とエピソードを追加します。
5. **本文を書く。** 中央のエディタに本文を入力します。
6. **保存する。** `Ctrl+S` を押すか、画面右上の「**保存**」ボタンをクリックします。

> 初回起動時にはガイドツアーが表示されます。サイドパネル・ビューモード・AI・コマンドパレットの位置を順に案内するので、最初に一度ご覧ください（「スキップ」でいつでも閉じられます）。

## 2. 画面の構成

StoryAttic は大きく「ホーム画面」と「ワークスペース」の 2 つから成ります。

### ホーム画面

起動直後に表示される、作品の一覧画面です。

- 作品はカード形式で並びます。カードをクリックするとその作品のワークスペースが開きます。
- 「**＋ 新規作成**」カードから新しい作品を作ります。
- 右上に「**AI Story Builder**」「**インポート**」ボタン、テーマ切替（太陽／月アイコン）があります。
- 各カードからは作品の削除もできます（削除は取り消せません）。

### ワークスペース

実際に執筆を行う画面で、次の領域で構成されています。

| 領域 | 位置 | 役割 |
|---|---|---|
| **ヘッダー** | 最上部 | ホーム／作品名／ビューモード切替／保存／検索／AI／出力／コマンドパレット |
| **アクティビティバー** | 左端の縦長バー | サイドパネルの切替（11 種類）と、文章分析・執筆支援・設定の起動 |
| **サイドパネル** | 左 | 選択中のパネル（目次・人物・プロットなど）の内容 |
| **エディタツールバー** | 中央上部 | 装飾・ルビ・縦書き・フォントサイズなどの編集操作 |
| **エディタ** | 中央 | 本文の編集領域 |
| **ステータスバー** | 最下部 | 文字数・行数などの情報、執筆タイマー、目標進捗 |
| **AI パネル** | 右（フローティング） | AI 思考パートナー（`Ctrl+Shift+A` で開閉） |

- ヘッダーの作品名をクリックすると、その場でタイトルを変更できます。
- 左端のアクティビティバーのアイコンをもう一度クリックすると、サイドパネルを閉じて執筆に集中できます。

## 3. プロジェクト（作品）の管理

ひとつの「プロジェクト」が、ひとつの作品に対応します。プロジェクトの中に「章」があり、章の中に「エピソード（話）」が入る三層構造です。

### 作品を作る

ホーム画面の「**＋ 新規作成**」カードからタイトルを入力して作成します。作成するとそのままワークスペースが開きます。

### 章とエピソードを編成する

- **目次パネル**（`Ctrl+1`）: 章とエピソードの一覧。クリックで本文に移動します。
- **章立てパネル**（`Ctrl+2`）: 章の追加・並び替え・構成の調整。
- 章・エピソードは**ドラッグ＆ドロップ**で並び替えできます。

### 作品を削除する

ホーム画面の作品カードから削除します。確認ダイアログが出ますが、**削除は取り消せません**。大切な作品は事前に JSON エクスポート（[第 9 章](#9-データの入出力)）でバックアップしておくことをおすすめします。

## 4. エディタの使い方

中央のエディタは ProseMirror ベースのリッチテキストエディタで、日本語小説の組版に対応しています。エディタ上部のツールバーから主な操作を行います。

### ツールバーの機能

| ボタン | 機能 | 補足 |
|---|---|---|
| 元に戻す／やり直し | 編集の取り消し・復元 | `Ctrl+Z` / `Ctrl+Y` |
| **B** / *I* | 太字／斜体 | `Ctrl+B` / `Ctrl+I` |
| ルビ | 選択文字にふりがなを振る | 文字を選択してから実行 |
| 傍点 | 選択文字に圏点（傍点）を付ける | 文字を選択してから実行 |
| 文字を小さく／大きく | フォントサイズ変更 | 10〜32px の範囲 |
| エディタ幅 | 本文の表示幅を切替 | 標準幅 → ワイド → 全幅 を循環 |
| 縦書き | 縦書き／横書きの切替 | |
| 校正ビュー | 誤用・冗長表現をハイライト表示 | [第 8 章](#8-文章分析校正執筆支援)参照 |
| 差分ビュー | 編集前後の差分を表示 | |
| 集中モード | サイドパネルの表示／非表示 | 執筆に集中したいとき |

### ルビの振り方

1. 本文中でルビを振りたい文字（漢字など）を選択します。
2. ツールバーの「ルビ」ボタンをクリックします。
3. ダイアログに読み（ふりがな）を入力し、「適用」を押します。

### 傍点（圏点）

強調したい文字を選択し、ツールバーの「傍点」ボタンをクリックすると、文字の脇に点が付きます。

### 保存について

- `Ctrl+S` またはヘッダーの「保存」ボタンで保存します。
- 未保存の変更があるときは、ヘッダーに「● 未保存」と表示されます。
- ホームに戻ろうとしたとき未保存の変更があれば、「保存して戻る／保存せずに戻る」を確認します。
- 自動保存の挙動は設定（「保存・データ」カテゴリ）で調整できます。

## 5. ビューモード

ヘッダー中央のビューモード切替で、本文をさまざまな見え方で確認・編集できます。

| モード | 内容 | ショートカット |
|---|---|---|
| **エディタ** | 通常の執筆ビュー | |
| **台詞一覧** | 「」『』（）内の台詞だけを抽出して一覧表示。クリックで本文の該当箇所へ移動 | `Ctrl+Shift+L` |
| **プレビュー** | 原稿用紙風（縦書き・マス目）またはスマートフォン風の表示で仕上がりを確認 | `Ctrl+Shift+P` |
| **デュアル** | 2 つのエピソードを左右に並べて同時に編集 | `Ctrl+Shift+D` |

> 台詞一覧・プレビュー・デュアルは、エピソードを開いているときに利用できます。デュアルビューでは `Ctrl+S` で両方のエピソードが同時に保存されます（同じエピソードを左右に開くことはできません）。

ツールバー側にある**校正ビュー**・**差分ビュー**も、本文の見え方を切り替える機能です（[第 8 章](#8-文章分析校正執筆支援)参照）。

## 6. サイドパネル — 物語の設計図

左端のアクティビティバーから、11 種類のパネルを切り替えられます。物語の設定や資料をここで管理します。`Ctrl+1`〜`Ctrl+0` で素早く切り替えられます。

| パネル | ショートカット | 内容 |
|---|---|---|
| **目次** | `Ctrl+1` | 章・エピソードの一覧。本文への移動 |
| **章立て** | `Ctrl+2` | 章構成の編集・並び替え |
| **人物** | `Ctrl+3` | キャラクターのプロフィール管理（タグ付け対応） |
| **プロット** | `Ctrl+4` | プロット構造・展開の整理 |
| **あらすじ** | `Ctrl+5` | 作品全体のあらすじ |
| **相関図** | `Ctrl+6` | 人物どうしの関係の整理・関係図 |
| **用語** | `Ctrl+7` | 用語集（固有名詞・設定語の定義） |
| **世界観** | `Ctrl+8` | 世界設定のマテリアル |
| **伏線** | `Ctrl+9` | 伏線の張り・回収を追跡 |
| **メモ** | `Ctrl+0` | 自由メモ（パネル内で直接編集可） |
| **情報格差** | — | 読者と登場人物の間の「知っている／知らない」を整理 |

- 多くのパネルは「一覧＋概要」を表示し、詳細の編集は各項目の「編集」ボタンからモーダルで行います。
- 編集モーダルは入力が止まると自動で保存されます。「キャンセル」を押すと、開いたときの状態に戻せます。
- **メモ**だけは例外で、パネル上で直接編集できます。

これらの設定情報は、後述の AI 思考パートナーに「コンテキスト（参照情報）」として渡すこともできます。

## 7. AI 思考パートナー「Catalyst」

StoryAttic の AI は「**Catalyst（カタリスト）**」という思考パートナーです。ヘッダーの「**AI**」ボタン、または `Ctrl+Shift+A` で右側にパネルが開きます。

### 基本の考え方 — 30% AI / 70% Human

Catalyst は、あなたの代わりに文章を書きません。**答えではなく「問い」を返す**ことで、あなた自身の発想と思考を引き出すことを目的としています。

- **代筆なし**: 完成した文章や続きの本文を提示することはありません。創作の主体は常にあなたです。
- **執筆中は沈黙**: 執筆フェーズでは AI は静かに見守り、呼びかけたときだけ応答します。

> アプリ内には Catalyst の詳しいマニュアルが用意されています。AI パネルからいつでも開けます。

### 創作フェーズ

いま物語のどの段階にいるかを AI に伝えると、支援のスタイルと参照する情報が自動で切り替わります。

| フェーズ | 段階 | AI のふるまい |
|---|---|---|
| **探索（Explore）** | アイデアを広げる | 「もし〜なら？」で可能性を広げる |
| **構造（Structure）** | 構成・プロット・伏線を固める | 矛盾や論理の整合性を問う |
| **執筆（Write）** | 書くことに集中する | 沈黙を守り、呼ばれたときだけ応答 |
| **改稿（Revise）** | 推敲・改善する | テーマ適合度・文体の一貫性を問う |

### 作家タイプ

あなたの思考スタイルを選びます（作品ごとに保存されます）。

- **Explorer（発散型）**: 可能性を広げたい人向け。発想を広げる問いを優先します。
- **Architect（構造型）**: 整合性を固めたい人向け。矛盾・因果・伏線回収など論理的な問いを優先します。

### 口調

AI の応答口調を「**丁寧**（既定）」「**カジュアル**」「**辛口**」から選べます。辛口でも「問いを返す」原則は保たれます。

### 作品の Core（核）

作品の中心を定義すると、AI はすべての問いをこの Core を意識して返します。

- **テーマ**: 作品全体を貫く主題（例: 赦しと再生）
- **中心感情**: 読者に最終的に残したい感情（例: 静かな希望）
- **作品の問い**: この作品が問いかける核心（例: 人は過去の自分を赦せるか）

入力が止まると自動保存されます。

### コンテキスト参照

AI に渡す参照情報を選べます。フェーズに応じて既定の組み合わせに自動で切り替わります。

| 参照ソース | 文字数上限の目安 |
|---|---|
| 本文（現在のエピソード） | 約 1,200 字 |
| 人物（キャラクタープロフィール） | 約 3,000 字 |
| 用語（用語集） | 約 2,000 字 |
| プロット | 約 2,000 字 |
| 世界観 | 約 1,000 字 |
| あらすじ | 約 3,000 字 |
| 伏線 | 約 5,000 字 |

### 深掘りモード（ソクラテス式）

AI が解決策を一切示さず、一問ずつ深い問いだけを返すモードです。

- 最大 5 往復（5 往復目で締めくくりの問いになります）。
- 複数質問や選択肢の提示はせず、開かれた問いのみ。
- 「終了」ボタンでいつでもリセットできます。
- 考えを深く掘り下げたいとき、行き詰まったときに有効です。

### 停滞検知（自動）

執筆フェーズ中のみ動作し、書き詰まりを自動で検知してバナーで知らせます。

- **アイデア停滞**: 10 分以上入力が止まったとき。
- **構造停滞**: 同じ箇所への繰り返し編集や大量削除が見られたとき。
- **モチベーション停滞**: タイピングのリズムが大きく乱れ続けたとき。

バナーの「問いを聞く」で即座に AI に送信、「無視する」で非表示にできます。

### クイックアクション

フェーズ × 作家タイプに応じたプリセットの問いです。クリックすると入力欄にテンプレートが挿入されるので、送信前に自由に編集できます。

### 対応 AI プロバイダーと API キー

OpenAI / Anthropic / Google Gemini / xAI に対応しています。利用するには設定画面（「AI」カテゴリ）で各社の API キーを登録します。API キーは Windows の資格情報マネージャー（Credential Manager）に安全に保管され、本文ファイルには書き込まれません。

## 8. 文章分析・校正・執筆支援

これらの機能は、左端アクティビティバー下部のアイコン、または**コマンドパレット**（`Ctrl+P`）から呼び出せます。

### 文章分析

現在のエピソードを多角的に分析します。次のタブで観点を切り替えて確認できます。

| タブ | 主な内容 |
|---|---|
| **構造** | 起承転結の推定、クライマックス位置（※参考指標）、文字数・文数など |
| **テンポ** | 文の長さのばらつき、台詞と地の文のパターン、場面転換 |
| **語彙** | 頻出語、語彙多様度（TTR）、五感（視覚・聴覚・触覚・嗅覚・味覚）の語彙分布 |
| **人物** | 登場人物の登場バランス |
| **感情** | 感情表現の分布・推移 |
| **物語** | 語り・地の文の傾向 |
| **文章** | 文体・リズム・語彙多様度などの総合指標 |
| **共鳴スコア** | AIによる4軸評価（緊張感・共感度・テンポ・意外性、各100点満点）＋改善提案 |

> 構造分析（起承転結の推定）はあくまで「参考指標」です。創作上の判断はご自身で行ってください。

### 校正ビュー

エディタツールバーの「校正ビュー」ボタン、またはコマンドパレットから切り替えます。誤用や冗長表現の候補を本文上でハイライトします。検出ルールは設定（「校正」カテゴリ）で調整できます。

### 差分ビュー

ツールバーの「差分ビュー」から、編集前後の変化を比較表示します。

### 執筆支援

執筆のモチベーションを支える機能群です。

- **執筆目標**: 目標文字数を設定すると、ステータスバーに進捗バーが表示されます。
- **執筆タイマー**: ステータスバーから 25 分などのタイマーを開始できます（ポモドーロ的な集中執筆に）。
- **執筆日記・履歴**: 執筆の記録を振り返れます。

## 9. データの入出力

すべてのデータはお使いの PC 内に保存されます。バックアップや他環境への移行には、以下の入出力機能を使います。

### ワークスペースの「出力」メニュー

ヘッダーの「**出力**」ボタンから選べます。

| 項目 | 内容 |
|---|---|
| **全データ JSON エクスポート** | 作品まるごと（本文・設定・資料すべて）を 1 つの JSON ファイルに書き出し。バックアップ・移行に最適 |
| **テキスト出力** | 全エピソードの本文をプレーンテキスト（.txt）で書き出し |
| **ZIP で一括出力** | エピソードごとのファイルをまとめて ZIP で書き出し |
| **テキスト取込** | 複数の .txt ファイルを読み込み、エピソードとして追加 |

出力ファイル名は「タイトル_日付_時刻」の形式で自動生成されます。

### ホーム画面のインポート

- **インポート**: JSON エクスポートしたファイルを読み込み、作品を復元・追加します。
- **AI Story Builder**: テキストや設定ファイルから、AI が物語の骨格（章・人物・設定など）を自動生成してインポートします。一から始める代わりに、素材から作品の土台を作りたいときに使います。

> 大切な作品は、定期的に「全データ JSON エクスポート」でバックアップを取ることをおすすめします。作品の削除は取り消せません。

## 10. 設定

アクティビティバー下部の歯車アイコン、またはコマンドパレットの「設定を開く」から開きます。設定は次の 5 つのカテゴリに分かれています。

| カテゴリ | 内容 |
|---|---|
| **執筆環境** | エディタ・原稿の見た目、テーマ（ライト／ダーク）など |
| **保存・データ** | 自動保存の挙動、ストレージ |
| **校正** | 誤用・冗長表現の検出ルール |
| **演出・サウンド** | 雰囲気の演出と環境音 |
| **AI** | AI プロバイダーの選択と API キーの登録 |

左の項目リストは上下キーでも移動できます。未保存の変更がある項目には印が付きます。

## 11. 演出・サウンド

執筆の雰囲気を高めるための機能です。コマンドパレットの「雰囲気設定」やステータスバーから操作できます。

- **パーティクル演出**: 雨・雪・桜などのエフェクトを背景に表示します。
- **環境音**: 雰囲気に合う環境音を再生します（音量は設定で調整）。
- **キャラクターウィジェット**: マスコットアシスタントを画面上に表示します。

執筆の妨げになる場合は、設定（「演出・サウンド」カテゴリ）やコマンドパレットからオフにできます。

## 12. キーボードショートカット一覧

### 基本操作

## 13. データの保存場所とプライバシー

- すべての作品データは、お使いの PC 内の `%APPDATA%\StoryAttic\` に SQLite データベースとして保存されます。クラウドには送信されません。
- AI の API キーは Windows の資格情報マネージャー（Credential Manager）に保管され、本文ファイルやエクスポートデータには含まれません。
- AI 機能を使うときのみ、入力内容と選択したコンテキストが、選んだ AI プロバイダー（OpenAI / Anthropic / Google / xAI）へ送信されます。送信される参照情報は、AI パネルのコンテキスト設定で選択・制限できます。
